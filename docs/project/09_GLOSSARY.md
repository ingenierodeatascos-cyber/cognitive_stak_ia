# GLOSSARY

Petit glossari per no perdre's amb els noms del sistema.

## Termes basics

### Change

Una unitat de treball concreta. Sempre hi ha un canvi actiu cada vegada.

### Proposal

Document que explica que es vol canviar i quin problema resol.

### Spec

Document que defineix requisits clars i verificables.

### Implementation report

Document que explica que s'ha implementat realment.

### Review

Validacio del canvi implementat.

### Archive

Tancament del canvi i pas a historic.

### Memory

Coneixement reusable que pot servir en futurs canvis.

### docs/memory

Memoria reusable consultable sota demanda.

### memory

Memoria interna del sistema. No s'ha de carregar per defecte.

### Orchestrator

Coordinador del flux. No implementa.

### Planner

Agent que delimita el canvi.

### Spec Writer

Agent que transforma la proposta en requisits.

### Implementer

Agent que executa la implementacio.

### Reviewer

Agent que revisa qualitat, riscos i desviacions.

### Archivist

Agent que tanca el canvi i filtra que val la pena recordar.
