# DEFINITION OF DONE

Aquest document defineix quan un canvi es pot considerar acabat.

No vol dir “sembla que funciona”. Vol dir que el canvi ha passat el nivell minim de qualitat del projecte.

## Un canvi esta fet quan

- te `proposal` clara
- te `spec` verificable
- la implementacio respecta el scope
- hi ha revisio
- no s'han detectat riscos critics sense visibilitat
- el resultat es pot explicar i verificar

## Checklist practic

### Abans d'implementar

- el problema esta ben definit?
- el scope esta tancat?
- la spec es prou clara?

### Despres d'implementar

- s'ha tocat nomes el necessari?
- hi ha un `implementation report` honest?
- queden tests pendents anotats?

### Abans de tancar

- el `reviewer` dona `PASS` o hi ha visibilitat clara dels problemes?
- el canvi es pot arxivar sense soroll?
- hi ha alguna decisio o patro que cal guardar a memoria?

## Regla curta

Si no pots explicar en 1 minut:

- que s'ha canviat
- per que
- com s'ha validat

llavors el canvi encara no esta realment tancat.
