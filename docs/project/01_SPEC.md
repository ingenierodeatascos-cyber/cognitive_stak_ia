# PROJECT SPEC

Aquest document defineix que es el projecte a nivell funcional.

No descriu un canvi concret. Descriu la intencio general del producte o aplicacio.

## Com omplir aquest document

Explica:

- quin problema resol el projecte
- a qui va dirigit
- quines funcionalitats principals tindra
- que queda fora per ara

## Plantilla recomanada

### Project summary

Resumeix el projecte en 2 o 3 frases.

### Problem

Quin problema real vols resoldre?

### Target user

Qui fara servir l'aplicacio?

### Core features

- funcionalitat clau 1
- funcionalitat clau 2
- funcionalitat clau 3

### Out of scope for now

- cosa que no entra a la primera versio
- funcionalitat futura

### Success criteria

- com sabras que el projecte te una primera versio valida?

## Exemple per a un estudiant de DAM

### Project summary

Aplicacio web CRUD per gestionar tasques personals de manera simple.

### Problem

L'usuari necessita una manera clara de crear, veure, editar i eliminar tasques sense eines complexes.

### Target user

Estudiant o usuari individual que vol gestionar les seves tasques.

### Core features

- crear tasques
- llistar tasques
- editar tasques
- eliminar tasques

### Out of scope for now

- treball col laboratiu
- notificacions
- integracions externes

### Success criteria

Un usuari pot gestionar les seves tasques en una app funcional, clara i estable.
