# Forma Recomanada de Treballar Setmana a Setmana

Aquest sistema funciona millor si treballes en blocs petits i repetibles.

## Rutina recomanada

### Inici de setmana

- revisa estat del projecte
- decideix quin es el canvi mes important
- assegura que sigui prou petit

### Durant la setmana

- treballa un canvi cada vegada
- segueix el flux base
- evita barrejar noves idees dins del mateix canvi

### Final de setmana

- revisa si el canvi es pot tancar
- arxiva si ja esta llest
- guarda nomes memoria reutilitzable

## Com dividir un projecte en canvis petits

Mala idea:

- “fer tot el sistema de login i panell privat i rols”

Bona idea:

- “afegir formulari de login”
- “protegir la primera ruta privada”
- “afegir logout”

## Com prioritzar el següent canvi

Tria el canvi que:

- desbloqueja el següent pas
- sigui petit
- tingui criteri de validacio clar

## Quan arxivar

Arxiva quan:

- la implementacio esta feta
- la revisio esta feta
- el canvi es pot explicar clarament

## Quan actualitzar memoria

Només quan hi hagi:

- una decisio reutilitzable
- un patro validat
- context estable

## Com no acumular soroll

- no obris molts fronts alhora
- no guardis idees temporals a memoria
- no facis canvis massa grans

## Idea final

Un bon ritme de treball no es anar rapid.

Es mantenir el projecte clar setmana rere setmana.
