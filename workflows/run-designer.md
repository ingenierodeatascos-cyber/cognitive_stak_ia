# RUN DESIGNER

## Quan s'ha d'utilitzar

Utilitza aquest workflow nomes si el canvi necessita una definicio visual o de UX abans d'implementar.

Si el canvi es tecnic o molt simple, mantingues el flux base i no l'activis.

Llegeix aquests fitxers abans d'actuar:

- AGENTS.md
- ORCHESTRATOR.md
- .atl/activation-matrix.md
- .atl/routing.md
- .atl/phase-policy.md
- agents/designer.md
- docs/active/current-change/02_spec_delta.md
- docs/project/01_SPEC.md
- docs/project/03_CONSTRAINTS.md
- skills/registry.md

I activa només la skill de frontend o UX si realment aplica.

## Missió

Definir l'estructura visual i l'experiència d'usuari del canvi actual.

No implementis codi.  
No redefineixis la SPEC.  
No entris en detalls tècnics interns innecessaris.

## Objectiu del resultat

Generar el contingut de:

- docs/active/current-change/03_design.md

## Regles obligatòries

- Clarifica pantalles, components o fluxos si cal
- Mantén coherència amb la SPEC
- Respecta constraints del projecte
- No afegeixis funcionalitats no demanades
- Prioritza simplicitat i claredat

## Format de sortida obligatori

Retorna només l'output estructurat definit a `agents/designer.md`.

No afegeixis introduccions.
No afegeixis explicacions fora del format.
